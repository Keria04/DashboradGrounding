📋 准备工作
══════════════════════════════════════════════════════════════════

✅ 检查Git是否安装:
   打开命令行，运行: git --version
   
   如果未安装:
   - 访问 https://git-scm.com/download/win
   - 下载并安装

✅ 检查Git LFS是否安装:
   运行: git lfs version
   
   如果未安装:
   - Git安装包已包含LFS，重新安装Git即可

══════════════════════════════════════════════════════════════════
🚀 5步上传流程
══════════════════════════════════════════════════════════════════

第1步: 在GitHub创建仓库（3分钟）⭐
────────────────────────────────────────────
   1. 访问 https://github.com/
   2. 登录（没有账号先注册）
   3. 点击右上角 "+" → "New repository"
   4. 填写:
      - Repository name: Project_srp
      - Description: 仪表盘图表检测系统
      - 选择: Public（公开）或Private（私有）
      - 不要勾选任何初始化选项
   5. 点击 "Create repository"
   
   获得仓库地址:
   https://github.com/你的用户名/Project_srp.git

第2步: 运行一键上传脚本（1分钟）⭐
────────────────────────────────────────────
   双击运行: 上传到GitHub.bat
   
   按照提示:
   1. 输入您的GitHub仓库地址
   2. 按回车（使用默认提交信息）

第3步: 配置Git LFS（自动）⭐
────────────────────────────────────────────
   脚本会自动配置Git LFS处理模型文件

第4步: 输入GitHub凭据（2分钟）⭐
────────────────────────────────────────────
   方式1 - Personal Access Token（推荐）:
   
   获取Token:
   1. GitHub → Settings（右上角头像）
   2. Developer settings → Personal access tokens
   3. Tokens (classic) → Generate new token
   4. 勾选 "repo" 权限
   5. Generate token → 复制token
   
   推送时输入:
   - Username: 你的GitHub用户名
   - Password: 粘贴刚复制的token（不是密码！）

第5步: 等待上传完成（5-10分钟）⭐
────────────────────────────────────────────
   上传内容:
   - 代码文件（约2MB）
   - 模型文件（21.5MB）- 使用Git LFS
   - 配置文件
   - 文档
   
   进度会实时显示
   
   看到 "✅ 上传成功！" 即完成

🔧 常见问题
══════════════════════════════════════════════════════════════════

Q: 推送时提示文件太大？
A: 确保Git LFS已安装并正确配置
   git lfs install
   git lfs track "*.pt"

Q: 推送失败，权限错误？
A: 使用Personal Access Token而不是密码
   Token获取: GitHub → Settings → Developer settings

Q: 推送速度很慢？
A: 模型文件21.5MB需要时间，耐心等待
   或使用国内镜像（Gitee）

Q: 不想上传模型文件？
A: 在.gitignore中添加:
   experiments/yolov8s_phase1_improved/weights/best.pt
   但这样别人就无法直接使用了
